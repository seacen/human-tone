# human-tone 设计说明

本文件就是最终设计，不是某个阶段性折衷。设计与实现 1:1 对应：这里写的每一层、每一条母模式、每一份文件，仓库里都能找到对应物。没有"完整版愿景"藏在别处，也没有"先凑合的 v1"。要改设计，就改这份文档连同它指向的文件。

## 一句话结论

用两层结构去除文字的"AI 味"：**通用层**只描述病的形状、怎么判、控制流，语言中立、零具体词；**语言侧**装每门语言的全部数据。加一门语言等于往 `languages/` 丢一个文件夹，通用层和脚本一行不改。去除为主、护栏作最后一道防线，精度优先于召回——误伤真人声口比残留 AI 腔更不可逆。

## 三面镜子（规则从哪来）

规则不是拍脑袋列的。三套各自定义了自己领域的语言学框架，分别对应设计的三个承重部位。哪面镜子管哪块，下面点明。

### 镜子一 · 奥威尔《政治与英语》（配 Strunk & White《The Elements of Style》）

**决定通用层装哪 16 条母模式。** 奥威尔那篇文章本身就是一份"去除清单"：死隐喻、虚假动词肢体、装腔措辞、空洞词、无生命的被动操作——每一类都是"生成过程带来、跨语言复现"的修辞病理，正好对应通用层要抓的东西。母模式里挂奥威尔的几条，都能在原文找到对应类：

- MP-06 万能动词空转 ← 虚假动词肢体（verbal false limbs）
- MP-10 名词化与被动偏好 ← 无生命的操作 / 被动语态
- MP-11 装腔大词 ← 装腔措辞（pretentious diction）
- MP-01 / MP-05 / MP-09（评价拔高 / 含糊对冲 / 极值判断）← 空洞词（meaningless words）
- **MP-12 死隐喻与拟物化、MP-13 比喻失控 ← 死隐喻（dying metaphors）**。把抽象事物拟成会"生长"或能"承托"的实体，正是奥威尔说的死隐喻被无意识地捡起来用；这两条挂奥威尔是对的，不是攀附。

其余八条（MP-02 机械对举、MP-03 罗列三段式、MP-04 叙述者导游、MP-07 戏剧化揭露、MP-08 总结书签、MP-14 戏剧化短句、MP-15 代号泛指、MP-16 冗余范畴壳）标"本稿新增"：它们是从"坏文风"直接操作化出来的，在奥威尔那里没有一一对应的命名类，所以不托他的名。`patterns.md` 里每条的 Orwell tie 列如实标注来源，避免把新增项冒充成经典。

### 镜子二 · 余光中《怎样改进英式中文》（配思果《翻译研究》《翻译新究》）

**决定汉语包装什么。** 余光中把欧化中文的病症拆得很细：的的不休、被字滥用、名词化泛滥、逐字直译腔。这些不是跨语言通用病，是汉语在翻译 / 生成压力下特有的走形，所以它们不进通用层，只进 `languages/zh/`：

- 名词化、无必要被动进 zh 包的 MP-10 实现行；
- "的的不休""被字滥用"这类母模式框不住的，进 zh 包 §④ 语言私有腔；
- 逐字直译（calque）不列为母模式（它是语言对特有的），改用 `precision.md` 的**反译判据**处理，对照表存 zh 包 §⑤（白空间→市场空白、引擎室→核心引擎等 7 条）。

### 镜子三 · Biber 语域多维分析 / Halliday 语域理论

**负责不误伤的精度骨架。** 同一个结构，在一种语域是 AI 腔，在另一种是规范表达："予以处罚"在公文里是固定搭配，"进行分析"在学术摘要里是常态，搬到自媒体短文里才是病。Biber 用多维分析证明语域之间的语言特征系统性地不同，Halliday 把语域讲成受场景约束的选择系统。结论直接落成机制：**先绑语域，再选规则**。`precision.md` 的语域门控、各语言包 §⑥ 的"语域带宽"、易混语域走交集，全部由这面镜子承重——没有它，高召回会直接变成高误伤。

三面镜子对应三个部位：镜子一决定通用层装什么，镜子二决定汉语包装什么，镜子三决定精度层怎么防止召回伤到规范表达。这就是"通用机制 + 每语言数据"分层的语言学依据。

## 通用 vs 语言特定：分界表

一条准则要么语言中立（进通用层），要么依赖某门语言的具体词（进包）。分界只问一句：**这句话里会不会出现某门语言的具体词？** 会，就进包。

| 内容 | 归属 | 为什么 |
|------|------|--------|
| 16 条母模式的形状、判定通则、Orwell tie | 通用层 `patterns.md` | 病的形状跨语言一致，写一次全语言受益 |
| 精度机制（语域门控 / 可替换性测试 / 密度阈值 / 白名单闸 / 反译判据 / 来源启发式 / 保守降级） | 通用层 `precision.md` | 都是语言中立的判断机制，落地词在包里 |
| 护栏类别、平庸化回滚症状、保留人味 | 通用层 `guardrails.md` | 类别抽象；具体术语词条在包 whitelist |
| 六步工作流、自查回路、双向打钩 | 通用层 `workflow.md` | 控制流与语言无关 |
| 语言发现 / 判定 / 混写路由 | 通用层 `resolver.md` | 机制枚举文件夹，不认识任何语言名 |
| 某语言的触发词、黑名单、私有腔、calque 表、语域带宽、白名单、宾语边界例 | 语言包 `languages/<code>/pack.md` | 全是该语言的具体词，只能出现在这里 |
| 该语言的机器可读检测规则与表层信号正则 | 语言包 `languages/<code>/signals.json` | 正则字面量依赖具体语言 |

通用层连英文触发词都不放：delve、tapestry、make a decision 这类英文例子归 `languages/en/`，不进 `patterns.md`。通用层用英文书写，只是拿英文当中性元语言，不等于允许英文实例。

## 通用加载机制（resolver）

加语言 = 丢文件夹，靠的是 `resolver.md` 描述的三步机制，机制里不写死任何语言名：

1. **发现**——枚举 `references/languages/` 下的子目录，有几个文件夹就支持几门语言。"现有 zh、en"只是对当前文件夹的陈述，不是机制依赖的常量。
2. **判定**——每个包的 `signals.json` 带 `detect` 块（CJK 占比、拉丁字母占比、虚词表），拿输入逐包打分选最高。新语言自带检测规则，resolver 不用改。
3. **路由**——中英混写按句拆分，逐句判语言、逐句加载对应包，让一段里的中文句和英文句各走各的母模式、白名单、calque 表。判不出的语言退回只用通用层，并提示未完全支持。

## 目录结构

```
skills/human-tone/                  # skill 包（目录名 = 调用名，npx skills add 装到此）
  SKILL.md                          # 薄入口（双语触发 description、六步、指针）
  references/
    patterns.md                     # 16 母模式目录（英文，零具体语言实例）
    precision.md                    # 精度七闸（英文，零实例）
    guardrails.md                   # 护栏 + 平庸化回滚 + 保留人味（英文，零实例）
    workflow.md                     # 六步详解 + 自查回路 + 双向打钩（英文，零实例）
    resolver.md                     # 通用加载机制（英文，零实例）
    languages/
      README.md                     # ★语言包协议（pack 长什么样 + 怎么加一门语言）
      zh/{ pack.md, minimal.md, signals.json }   # 结构化数据 + 一页浓缩规则 + 正则
      en/{ pack.md, minimal.md, signals.json }
  scripts/
    check.mjs                       # 通用信号复扫器（Node≥18、零依赖、只报告）
install.mjs                         # 跨 agent 安装器（npx skills 装 skill + 往各配置写引用）
README.md / README.en.md            # 中文默认 + 英文
docs/
  DESIGN.md                         # 本文件
  research/                         # 外部 9 skill 调研

```

分层遵循 Skill Creator 约定：按需加载文档进 `references/`，脚本进 `scripts/`，不设顶层 `core/`、`lang/`。SKILL.md 是薄入口，重内容按需披露——对抗调研里看到的体量陷阱（900+ 行的 skill，作者自己都担心模型执行不全）。

## 16 母模式目录

语言中立，ID 钉死，全仓库一致引用。判定通则贯穿全部：**单次出现不判**，只惩罚短段内成堆、脱离内容、删掉不丢意的；挂"奥威尔"的能在《政治与英语》找到对应类，挂"本稿新增"的是从坏文风操作化出来、不托奥威尔之名。

| ID | 名称 | 来源 | 形状 |

| MP-01 | 评价拔高 | 奥威尔·空洞词 | 给平常事贴不带信息的价值标签 |
| MP-02 | 机械对举 | 本稿新增 | 否定—肯定 / 对立项对仗撑气势而非表意 |
| MP-03 | 罗列三段式 | 本稿新增 | 三项以上凑数排比，删一项不丢意 |
| MP-04 | 叙述者导游 | 本稿新增 | 叙述者出面领路的元话语 |
| MP-05 | 含糊对冲 | 奥威尔·空洞词 | 不承担判断的对冲、模糊限定 |
| MP-06 | 万能动词空转 | 奥威尔·虚假动词肢体 | 轻动词 + 名词吞掉一个实义动词 |

`patterns.md` 里每条另给英文规范名、shape、fix，全部英文、零具体语言触发词。具体到"哪些词、哪种严重度、哪个语域豁免"，进各语言包的母模式实现表。calque 不在此表——它按精度判据处理（见下）。

## 语言包格式

一门语言的全部数据分两份，`pack.md` 给人读、`signals.json` 给机器读。

`pack.md` 八段：① manifest（语言码 / 名 / 识别特征 / 维护戳）② 母模式实现表（MP-ID | 触发词或句式壳 | 严重度 | 语域豁免，只填 delta）③ 黑名单（按问题族，代表词非穷举，每族注"变体一并禁，别换近义词逃逸"）④ 语言私有腔（母模式框不住的，中文如的的不休、被字滥用）⑤ calque 对照表（仅目标语言方向；英文作源语言时此表空）⑥ 语域带宽（各语体哪些 MP 不触发）⑦ 白名单（术语 / 公认缩写 / 成分 / 专名 / 有意排比标记）⑧ 宾语边界例（可替换性测试的正反语料）。

`signals.json` 结构固定：`{ "lang", "displayName", "detect": {…}, "signals": [ { "id", "label", "regex", "flags" }, … ] }`。正则用字符串存，`check.mjs` 用 `new RegExp` 读。每语言给 6–9 条可正则化的表层信号——只抓能正则化的浅层形状，语域、密度、真人声口交给模型。

## check.mjs 通用化

脚本定位是**确定性复扫补召回**：模型改完，用正则再扫一遍表层信号，把召回从"靠模型记性"变"机器补扫"。关键约束：

- **代码零语言字面量。** 用 `readdir` 枚举 `languages/*/signals.json`，所有触发词、正则、阈值全从包里读。加一门语言不用碰脚本。
- **零依赖、Node≥18。** 只用内置 `fs` / `path`。单个包坏了（缺文件 / JSON 错 / 正则编不出）只跳过它并提示，不拖垮其它语言。
- **只报告、不改写。** 输出命中的行号 / 母模式 / 命中文本。命中 ≠ 该改：正则抓不到语域、密度、真人声口。判断权在模型，冲突以模型为准，且因脚本提示改了什么须留痕。
- 用法 `node check.mjs <file> [--lang code]`，`--lang` 跳过自动判语言。

## 六步工作流 + 精度七闸 + 召回双路径

**六步**（`workflow.md` 详解）：① 识别语言与来源（真人手稿抬高门槛）② 语域门控（先绑语域再选规则，易混走交集）③ 母模式扫描（按密度，单次不判）④ 减法改写（能删就删，删不掉才改短 / 还原实义动词 / 换地道说法；保护跨度字节不动）⑤ 自查复扫 ⑥ 输出（默认终稿，用户要标注才给对照）。

**精度七闸**（`precision.md`，每个候选改动都要过）：语域门控 → 可替换性测试（把壳换成素形，意思还在就是填充，删）→ 密度阈值（看短段不看单 token）→ 白名单硬闸（术语 / 缩写 / 专名命中即移出候选）→ 反译判据（判 calque：反译回源语言才通顺 = 病）→ 来源启发式（真人手稿抬门槛，来源轴 ≠ 语域轴）→ 保守降级（过完仍拿不准就挂起标 `[需复核]`，不猜着删）。

**召回双路径**：一条是模型语义改写 + 强制自查回路（改完列 ≥2 条残留再改一版、≤3 轮、仍存疑标 `[需复核]`），另一条是 `check.mjs` 确定性正则复扫补召回。两条路径加双向打钩（原文每条事实终稿要有、终稿每条事实原文要有，防漏也防增），召回和保真同时钉住。

## Rubric 与三档对比测试

- **Rubric**（`evals/rubric.md`）五维打分，做成可持续优化的 Goal：召回 / 精度不误伤 / 信息保真 / 自然度不平庸化 / 声口保留。误伤是硬扣分项——精度优先于召回在评分口径里也钉死。
- **三档对比**（`evals/comparison.md`）：A 裸（无 skill、无 user CLAUDE.md）暴露原始 AI 味基线；B 仅 user CLAUDE.md 规则暴露单靠规则能去多少；C 用 human-tone skill 暴露在 B 之上的增量（精度护栏 + 召回双路径 + 多语言 + 脚本）。按 rubric 打分对比。
- **cases**（`evals/cases/`）：中英各若干，覆盖自媒体 / 商务邮件 / 技术文档 / 公文 / 学术 / 口语，含至少 2 个纯精度陷阱（几乎不该改的真人 / 规范文本），验证不误伤。对应调研里的 SF/SNF 双集：SF 量召回，SNF 量精度。
