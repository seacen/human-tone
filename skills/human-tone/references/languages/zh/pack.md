# 中文语言包（zh）

本文件是中文的**全部数据**，纯数据、无判据。判据（怎么绑语域、怎么算密度、怎么兜底）在通用层
`references/precision.md`、`patterns.md`、`guardrails.md`、`workflow.md`；这里只回答一件事：
通用层那 16 条母模式，在中文里**长什么样、用哪些词、哪些语域该放过**。母模式 ID（MP-01..MP-16）
与通用层严格对齐，改这里不改那边的编号。

英文包（`en`）与本文件是同构的姊妹文件：结构、段号、术语一一对应，只是里头的词换成各自语言的。

---

## ① manifest（语言标识）

- 语言码：`zh`
- 语言名：中文（简体为主，繁体触发词同族一并适用）
- 识别特征：CJK 汉字占比高；无空格分词；虚词「的、了、和、是、在、这、那」高频；标点用「，。、；：」。
  机器判语言规则见 `signals.json` 的 `detect`（CJK 占比阈值）。
- 维护戳：2026-07-06

## ③ 黑名单（按问题族，代表词非穷举）

**变体一并禁，别换近义词逃逸。** 下列每族给代表词，同族的近义、缩写、拆分写法（如「赋 能」「赋-能」）
一律同等对待——换个近义词、拆开写、加个后缀，都算同一族，不放过。命中不等于必删：先过语域门控
（§⑥）和白名单（§⑦），再决定删/换/留。

- **装腔黑话族**（→ MP-11）：赋能、抓手、闭环、落地、颗粒度、拉通、对齐、心智、顶层设计、底层逻辑、
  方法论、组合拳、护城河、生态位、价值主张、认知升级、降维打击、第一性原理、飞轮、势能。
  换地道说法直说做什么（赋能→帮 / 让…能；抓手→办法 / 着手点；落地→做出来 / 用起来；对齐→统一口径 / 达成一致）。
- **评价拔高族**（→ MP-01）：无缝、丝滑、极致、完美、优雅、强大、丰富、卓越、深刻、精彩、
  匠心、革命性、颠覆性。删标签，换成它到底改变了什么。
- **套话族**（→ MP-08）：综上所述、总而言之、总的来说、由此可见、值得一提的是、众所周知、
  不难看出、在当今…的时代、随着…的不断发展、在…的大背景下。开场收尾套话直接删。
  **原创格言式升华收尾**（给全文缀价值标签、零新信息）也算 MP-08——收在最后一个具体句、
  删升华壳（靠删壳/稀释，别整删有信息的句子）。
- **万能动词族**（→ MP-06）：进行、作出、加以、予以、给予、实现、达成、开展、推进（当轻动词空转时）。
  还原实义动词（进行讨论→讨论；作出决定→决定；给予支持→支持）。
- **极值族**（→ MP-09）：史无前例、前所未有、无与伦比、彻底、颠覆、革命性地。无据的绝对化删或降级。

## ④ 语言私有腔（母模式框不住的中文特有病）

这几类是汉语独有、通用母模式抓不到的，靠余光中《怎样改进英式中文》操作化出来。

- **的的不休**：一句里连挂多个「的」，修饰语层层叠「的」。改法：删可省的「的」，或拆句、换语序。
  例：「一个具有高度的可复用性的的组件」→「一个可复用的组件」。
- **被字滥用**：无必要的被动「被」，以及「被…所…」双重被动。中文常用主动或无标记受事。
  改法：改主动或去「被」（「这个问题被解决了」→「这个问题解决了」；「被大家所熟知」→「大家都知道」）。
- **「一个」滥用**：仿英文不定冠词，处处加「一个」。改法：多数可删（「这是一个很好的例子」→「这是很好的例子」）。
- **抽象「性 / 化」后缀堆叠**：可读性、可维护性、系统化、常态化 连用撑抽象。改法：能还原成动词/形容词就还原
  （「提升了系统的可维护性」→「系统更好维护了」）。
- **「们」滥用**：给无需复数的名词加「们」（「读者们」「用户们」）。改法：多数删。
- **「当…的时候」欧化时间从句**：仿英文 when 从句前置。改法：用「…时」或直接叙述
  （「当他离开的时候」→「他离开时」）。
- **「关于 / 对于 / 就…而言」前置累赘**：仿英文介词短语开头空转。改法：并入主句或删。

## ⑤ calque 对照表（英译中翻译腔）

判据在通用层 `precision.md` 的翻译腔反译判据：**把中文反译回英文才通顺、直接读中文别扭 = 病**。
下表左列是逐字直译（calque），右列是地道说法。中文是目标语言，源语言是英文（括注还原）。

核心 7 条（锁定，必收）：

| 直译腔（calque） | 地道说法 | 源语言还原 |
|------------------|----------|-----------|
| 白空间 | 市场空白 | white space |
| 引擎室 | 核心引擎 | engine room |
| 价格洞 | 价格缺口 | price hole |
| 头顶空间 | 余量 | headroom |
| 扭结 | 拐点 | kink |
| 梯子 | 阶梯 | ladder |
| 深潜 | 深挖 | deep dive |

另收几条常见的（超集扩展，同一反译判据）：在一天结束时→说到底（at the end of the day）、

## ⑥ 语域带宽（各语体的基线与豁免）

同一个句式，在不同语体里是规范还是 AI 腔。先绑语域再选规则（判据见 `precision.md` 语域门控）。
下表给中文各语体把哪些 MP 的基线抬高（更宽容）；没列到的语体走标准阈值。

- **公文 / 政务**：MP-08 套话（固定行文套语如「为…提供…保障」）、MP-06「予以/加以」法条搭配、
  MP-10 名词化——基线抬高，属规范。但 MP-11 黑话、MP-12 拟物、MP-02 空对仗照判。
- **学术**：MP-05 审慎对冲（「可能」「在一定条件下」）、MP-10 高名词化——基线抬高。MP-01/MP-09 照判。
- **商务邮件**：MP-08 客套开头结尾（「感谢您的…」「期待回复」）——基线抬高。MP-11 黑话照判（商务黑话是重灾区）。
- **自媒体 / 营销**：MP-01 评价词、MP-09 最高级——允许一定密度，成堆才判；MP-14 短句是风格，成堆造势才判。但 **MP-03 结构性排比与 MP-08 原创升华在劝说文里不豁免**——通篇齐整三拍 + 格言收尾正是这类文的 AI 重灾区，按 `patterns.md` 稀释处理（保信息、破节奏）。
- **网文 / 口语**：MP-14 断裂短句、MP-16 口语「…这种东西」——基线抬高，属风格。
- **技术文档**：MP-10 名词化部分规范；MP-11 里的真术语（闭环、颗粒度在特定技术语境）走白名单，见 §⑦。

## ⑦ 白名单（别改）

命中黑名单也别动的情形。白名单是硬闸门，优先于黑名单（判据见 `guardrails.md`）。

- **公认缩写**：RPI、SKU、NRR、GMV、ROI、KPI、OKR、API、SaaS、DAU、MAU、UV、PV —— 保留，别翻译别展开。
- **真术语**：黑名单词落在其**本行术语语境**时保留。「闭环」在控制系统 / 反馈回路是术语；
  「颗粒度」在数据粒度语境是术语；「对齐」在排版 / 内存对齐是术语。靠语域区分（§⑥），别一刀切。
- **专名 / 产品名 / 品牌名**：原样保留。
- **成分 / 剂量 / 型号 / 版本号**：原样保留。
- **引文**：引号内、参考文献、他人原话——不改。
- **有意排比标记词**：仅限作者**显式标注**的仿写、引用、戏仿（如「借用某某的说法：…」后的对仗）——不算 MP-02。**无标注的通篇 / 高密度排比不自动豁免**，走 MP-03「结构性重复」判据（`patterns.md`：稀释而非整删，别默认落到保留）。

## ⑧ 宾语边界例（可替换性测试正反语料）

供 MP-06 万能动词消歧用（测试法在 `precision.md`）：轻动词后的名词能不能拆成实义动词，
拆得通 = 空转该改；拆不通 = 固定搭配别动。

**该改（正例，轻动词空转）：**
- 进行讨论 → 讨论
- 作出决定 → 决定
- 给予支持 → 支持
- 加以说明 → 说明
- 实现增长 → 增长（多数场合）

**别改（反例，固定搭配 / 实义 / 语域规范）：**
- 进行手术 —— 「手术」不能当动词拆（不能说「手术它」），保留。
- 进行曲 —— 固定词，非「进行 + 曲」。
- 予以处罚 / 予以驳回 —— 公文法条固定搭配，属语域规范，保留。
- 作出贡献 —— 已是最自然说法，无更实义的替换，保留。
- 给予厚望 —— 固定搭配，保留。
