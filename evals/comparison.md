# 三档对比测试 — skill 到底加了什么

一句话目的：把"去 AI 味"这件事拆成三档跑同一批 case，看清 **skill 相对于"只在 CLAUDE.md 里写几条规则"到底多做了什么**。差值就是答案，不靠感觉。

## 三档怎么设，各自暴露什么

同一份源文，喂给三种配置，其余条件（模型、温度、指令措辞）保持一致。区别只在"手里有什么规则"。

| 档 | 配置 | 拿到手的规则 | 这一档暴露什么 |
|----|------|------------|--------------|
| **A · 裸** | 模型原样，无 skill、无 user CLAUDE.md 的去味条款 | 只有一句"把这段改得像人写的" | **原始 AI 味基线**：不给任何规则时模型自己能去掉多少。也顺带暴露模型自己就爱产的 AI 腔。 |
| **B · 仅 CLAUDE.md** | 把 `skills/human-tone/references/languages/<lang>/minimal.md` 粘进 CLAUDE.md，无 skill | 一页极简规律 + 护栏 | **一页规则单独的力**：不上完整 skill，光靠粘一页规律能去多少、护栏够不够。 |
| **C · 用 skill** | 加载完整 human-tone skill | 16 母模式 + 精度骨架 + 语域门控 + 语言包 + 可选 check.mjs | **skill 在 B 之上的增量**：多语言语言包、语域门控、白名单闸门、宾语消歧、密度阈值、自查回路、脚本复扫各自兜住了 B 漏掉或误伤的地方。 |

读数的方向固定：**A→B 的差 = 一页 CLAUDE.md 规则的贡献；B→C 的差 = 完整 skill 的贡献。** 把两段增量分开看，才知道复杂度买回了什么——若 B→C 几乎没差，说明 skill 相对一页规则没赚到复杂度，该回去砍；若 B 在精度陷阱上翻车而 C 兜住了，说明护栏骨架是真值钱。

为什么要有 B 这一档、不直接 A 对 C：没有 B，就分不清 C 的好是"规则本身"还是"skill 的机制"带来的。B 把"规则内容"这个变量单独拎出来，剩下 A→B 与 B→C 两段各自干净。

## 运行步骤

1. **选 case**：从 [`cases/`](cases/) 取样本，中英都要覆盖，且必须含精度陷阱样本（`*-trap`）——只跑高召回样本会让 A 档虚高、掩盖误伤。
2. **三档各跑一遍**：每个 case 在 A、B、C 三种配置下各改写一次，存下三份终稿。三档用同一句触发指令，别给 C 额外提示。
3. **按 rubric 打分**：对每份终稿按 [`rubric.md`](rubric.md) 的五维各打 0/1/2，记进下方表。精度（维度 2）或信息保真（维度 3）为 0 的，整份标"不合格"，不看总分。**双模型交叉**：打分环节换第二个模型把同一批终稿复打一遍，只采纳两模型一致的分；分歧的 case 人工裁一次。这一步消掉"改写模型自己给自己打高分"的单模型偏置，跟精度陷阱一样是防自欺的门。
4. **算增量**：同一 case 的 C 总分减 B 总分、B 减 A，填进增量列。三档同分的 case 要留意——要么源文太浅（换难样本），要么 skill 在这类文本上确实没加价值（记下来）。
5. **看陷阱档位**：精度陷阱样本单独看——期望是三档都尽量别改，重点看 A/B 会不会误删、C 有没有靠护栏守住。这里 C 的价值体现在"没动"，不在"改得多"。

## "表达差异到底在哪"怎么呈现

打分给的是量，但用户真正想看的是**质**：加 skill 前后，同一句话最后长什么样。用三栏并排 + 逐处标注，一屏看清。

**三栏并排**（同一段源文，三档终稿对齐放）：

```
源文  | 在这个瞬息万变的时代,我们的产品不仅仅是一个工具,更是一种理念。
A     | 在快速变化的时代,我们的产品不只是工具,更是一种理念。      ← 只换了同义词,套话骨架还在
B     | 我们的产品能帮你做 X。                                    ← 套话删了,但把"理念"一并删可能丢了作者本意
C     | 我们的产品能帮你做 X。(问:"理念"若是作者真要表达的定位,标[需复核]) ← 删得准且标出存疑处
```

**每处改动挂一个标签**，让差异可追溯，不是"感觉 C 更好"：

- 删了什么母模式（挂 MP-ID，如 `MP-08 总结书签`、`MP-02 机械对举`）。
- 守住了什么护栏（如 `保留:公文套语`、`保留:引文原文`、`保留:真人声口"我赌它撑不过一年"`）。
- 哪处 A/B 误伤了而 C 没有（精度差异），或哪处 A/B 漏了而 C 去了（召回差异）。

**收一句增量解读**，直接回答用户的问题：这个 case 上，一页 CLAUDE.md 规则（B）已经能去掉哪些，skill（C）额外兜住的是——多语言语言包认出的本语言特有腔（如中文的的的不休、英文的 delve/tapestry）、语域门控挡下的规范表达、白名单守住的术语引文、自查回路补掉的残留。把"额外兜住的"逐项点名，用户就知道复杂度花在哪了。

## 一起看的两张总表

跑完全批 case，汇两张表，别只看单 case：

- **增量表**：各 case 的 B−A 与 C−B 两列并排。看 C−B 的分布——集中在精度陷阱和多语言样本上，说明 skill 的价值正落在"护栏 + 语言包"这两处设计承重上，符合预期；若 C−B 普遍为 0，回去质疑 skill 是否值得这套复杂度。
- **陷阱表**：所有 `*-trap` 样本的精度维得分。这张表是 skill 的底线体检——A/B 在陷阱上掉分、C 守住，才证明"偏精度、误伤比残留更不可逆"这条锁定前提被机制真正落实了，而不只是写在文档里。
