# human-tone 成功标准（高门槛）

分两块:**触发**（自动发生）和**质量**（用了确实变好）。两块都达标才算成。任一未过,回去改 skill 或触发机制,不改标准。

## A. 触发 — 分两条路（一条自动、一条按需）

**官方实测（skill-creator `run_eval`，`evals/trigger-eval.json` 20 条 × 2 次）**:内容生成任务（公众号/小红书/PPT/邮件/官网文案）触发率 **0%**；非内容任务 100% 正确不触发。这不是 description 不够好——模型对"自己能写的任务"根本不去调 skill，**任何 agent 都一样**，是机制天花板，不可移植的 hook 才能强拦，但那违背"任何 agent 可用"，故弃。

所以"自动去味"不靠 skill 自触发，分两条路达标：

- **自动（跨 agent）= 极简规则常驻配置**：把 `skills/human-tone/references/languages/<lang>/minimal.md` 放进 agent 的 `CLAUDE.md` / `.cursorrules` / `AGENTS.md` / system prompt（装法见 `README.md` 与 `install.mjs`）。达标线：放进配置后，同样的内容生成任务，产出的 AI 味母模式显著下降（`check.mjs` 残留大幅降、盲评更像人写）。常驻规则是指令不是可选工具，可靠性远高于 skill 自触发。
- **按需（高精度）= 完整 skill**：用户显式要"去 AI 味 / 改得像人写"，或 agent 主动调起时触发。达标线：`trigger-eval.json` 里"显式去味类"请求命中率高、非内容任务 0 误触发；质量见 B。

## B. 质量 — 用了 skill 之后确实变好

每个测试样本,with-skill 版对标 `evals/rubric.md`（5 维 × 0-2 = 满分 10):

- **召回** ≥ 高:AI 味母模式清干净（`check.mjs` 残留 ≈ 0,或残留是语域豁免/白名单的合理放行);
- **精度 = 硬门槛**:**零附带损伤**——不丢任何事实/数据/专名/链接,不抹作者声口,不误伤语域规范表达,不平庸化。任何一处误伤即不达标;
- **保真**:事实不丢不增;
- **自然度 + 声口**:像该场景该有的真人写的,作者立场/语气/判断都在;
- **总分 ≥ 9/10**,且盲评更偏 with-skill;召回明显优于 baseline,而精度/保真/声口不倒退。

## C. 三档对比 — 验证增量来自哪

见 `evals/comparison.md`:**A 裸**（无 skill、无 CLAUDE.md 规则)/ **B 仅 CLAUDE.md 规则** / **C 用 skill**。

- C 相对 B 要有**可见增量**（精度护栏 + 多语言 + 更全的 16 母模式 + 脚本兜底);
- C 在**精度陷阱样本**（公文/学术/术语/真人声口)上 **0 误伤**,证明"更狠地去味"没换来"更多误伤"。

## D. 触发场景端到端实测

真实任务走一遍:"帮我做个 PPT / 写篇公众号 / 写个小红书文案",确认三件事同时成立——**自动触发** + 产出**确实去了 AI 味** + **没伤内容**。PPT 场景还要确认去味只作用于文字、不破坏结构与设计系统。
